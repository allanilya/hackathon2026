**GSAP fade-in animation:**
- Add a ref to the hero section: `const heroRef = useRef<HTMLElement>(null)`
- Use useGSAP with gsap.matchMedia() for reduced motion support (EXACTLY as shown in RESEARCH.md Pattern 4)
- For no-preference users: animate children elements (headline, subline, form, visual) with staggered fade-in
  - Use gsap.from on multiple targets with stagger: 0.15
  - autoAlpha: 0, y: 30, duration: 0.8, ease: "power2.out"
- For reduced-motion users: simple autoAlpha: 0 -> 1, duration: 0.3, no y movement
- Cleanup: return () => mm.revert()
- Scope: { scope: heroRef }

**Important notes:**
- Do NOT animate the LCP element with a long delay (headline should appear quickly)
- Keep stagger values small (0.1-0.15) so content appears fast
- The hero is a client component (needs "use client") for GSAP hooks
- Email input state is local useState only — NO form submission logic, NO API calls
- On form "submit" (if user presses Enter), just preventDefault and do nothing
